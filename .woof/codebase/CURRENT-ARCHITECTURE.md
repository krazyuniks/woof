# Current Architecture

## Layered topology

Woof is structured in four layers (ADR-001):

- **State layer.** Files under `.woof/` are the canonical, durable record. Schema-governed TOML config, JSON artefacts, JSONL audit logs, and markdown gate/plan/disposition files. No in-memory state survives a crash; the runner reconstructs position from disk on every invocation.
- **Engine layer.** Pure Python at `src/woof/`. Deterministic graph transitions (`src/woof/graph/`), schema validation, typed audit and event writes, the in-process `run_graph` runner, and the Stage-5 check matrix (`src/woof/checks/`). The engine does not perform inference; it dispatches, verifies, and commits.
- **Operator skill layer.** Claude Code skills under `skills/`. The `/woof` umbrella skill maps operator requests to `woof` CLI calls. `/woof:brainstorm` is the only specialist; it owns the interactive design phase. No other specialist skills exist.
- **Dispatched layer.** Stateless, isolated LLM subagents spawned by graph node handlers. Claude (`claude` CLI) serves as the Stage-5 executor (has LSP). Codex serves as the Stage-5 reviewer and the default primary for planning stages.

## Graph stages

Five stages run in fixed sequence. `next_node` in `src/woof/graph/transitions.py` drives stage routing from on-disk artefact presence.

- **Stage 1 — Discovery.** Four buckets: `discovery_research`, `discovery_thinking`, `discovery_ideate`, `discovery_synthesis`. Each bucket completes once its output artefacts exist. An optional brainstorm bundle (from `/woof:brainstorm`) can substitute for the headless research/thinking/ideate chain.
- **Stage 2 — Definition.** `epic_definition` produces `EPIC.md` with YAML front-matter (`observable_outcomes[]`, `contract_decisions[]`, `acceptance_criteria[]`).
- **Stage 2.5 — Contract readiness.** Deterministic. `contract_readiness` evaluates the Stage-2 contract for machine-checkable criteria, concrete decisions, resolvable refs, and decomposition sufficiency. Produces `readiness-result.json`; opens a `readiness_gate` on failure.
- **Stage 3 — Breakdown / Plan.** `breakdown_planning` produces `plan.json`. `plan_critique` produces `critique/plan.md`.
- **Stage 4 — Plan gate.** Deterministic. Opens `gate.md` from plan critique. Operator resolves with `approve`, `revise_epic_contract`, `revise_plan`, or `abandon_epic`.
- **Stage 5 — Story execution.** Per story: `executor_dispatch` → `critique_dispatch` → `review_disposition` → `verification` (9 checks) → `commit`. Gate halts on blocker or check failure. The graph re-enters on `woof wf --epic N`.

## Runner contract

`woof wf` holds a per-epic workflow lock (`.woof/epics/E<N>/.wf.lock`) while running. The runner calls `next_node`, executes the returned handler, and loops until a gate opens, a `StageStateError` halts, or the epic completes. `--once` exits after one node. `run_graph` catches `StageStateError(operator_recoverable=True)` from both `next_node` and node handlers and converts them to `gate_opened` events rather than crashing.

## Dispatch supervision

Dispatch-shaped nodes spawn a public-CLI subprocess via `woof dispatch` (ADR-008). Supervision uses phase-scoped clocks: idle and wall-clock timeouts before a terminal marker, completion-grace and tail-cap after. The `exit_type` field in `executor_result.json` carries the outcome; `completed_lingering` is a successful finish, not a timeout.

## Gate model

Gates are schema-validated `gate.md` files. `gate.md` presence in an epic directory short-circuits `next_node`. The operator resolves via `woof wf --epic N --resolve <decision>`. `plan_gate` and `story_gate` are the two gate types. `incomplete_stage_state` halts (cartography-missing or malformed state) use `gate_type` matching the node's stage.

## Cartography

The `.woof/codebase/` artefact group is mandatory (ADR-004). Three layers:

- Design: `TARGET-ARCHITECTURE.md`, `PRINCIPLES.md` — human-authored.
- AS-IS: seven mapper-authored markdown documents (this document and its siblings).
- Mechanical: `tags` (ctags), `files.txt` (git ls-files), `freshness.json` (staleness stamp) — gitignored, regenerated by `scripts/refresh-cartography`.

Preflight enforces presence and non-stub state of the design layer docs, presence and executability of the refresh script, and presence of the mechanical layer files.
