# ADR-004: Cartography Prerequisite

Superseded by ADR-013. Cartography remains first-class, but the required cartography floor is declared by repo policy instead of being mandatory for every run.

## Context

LLM-dispatched nodes pay tokens to rediscover repo structure when prompts arrive cold. Without a canonical, scoped, on-disk codebase index, every dispatch repeats the same exploration via shell tools in the agent loop. A small, fresh, structured artefact group eliminates that overhead by giving every node prompt-ready context cheaply.

## Decision

Every consumer repo has a mandatory cartography artefact group at `.woof/codebase/`, in three layers.

### Design layer (human-authored, durable)

Authored by the operator during setup. Refreshed only when architectural strategy changes.

- `TARGET-ARCHITECTURE.md` — the shape the project is designed to be. Greenfield: written before code exists. Brownfield: the operator chooses pragmatic acceptance (target equals current), aspirational refactor (target is the goal), or mixed.
- `PRINCIPLES.md` — cross-cutting design principles.

### AS-IS layer (mapper-authored)

Seven themed markdown documents produced by parallel mapper subagents that explore the codebase and write directly to disk. Refreshed on demand through the `/woof` map-codebase flow.

- `CURRENT-ARCHITECTURE.md` — observed patterns, layers, data flow, abstractions, entry points.
- `STACK.md` — languages, frameworks, runtime, dependencies.
- `INTEGRATIONS.md` — external APIs, databases, auth, monitoring, CI/CD.
- `STRUCTURE.md` — directory layout, where to add new code, naming.
- `CONVENTIONS.md` — naming, formatting, imports, comments, function design.
- `TESTING.md` — framework, file organisation, mocking, fixtures, coverage.
- `CONCERNS.md` — tech debt, known bugs, security risks, fragile areas.

### Mechanical layer (post-commit refreshed)

Regenerated by the post-commit hook via the consumer-owned `scripts/refresh-cartography`.

- `tags` — ctags universal index.
- `files.txt` — `git ls-files` output.
- `freshness.json` — `{ts, git_ref, age_s, generator_version}`.

ADR-009 adds a planned structural-index extension under `.woof/codebase/structural/`.
That extension enriches the mechanical layer with queryable files, symbols, and typed
edges, but it does not replace the baseline files above.

### Refresh rule

- Mechanical layer: every commit, via post-commit hook.
- Mapper docs: on demand through `/woof`. Staleness check at epic start with operator prompt. No automatic mid-epic remap.
- Design layer: human-driven.

### Preflight enforcement

`woof preflight` fails closed on:

- Missing `scripts/refresh-cartography` (or present but non-executable).
- Missing or stub `TARGET-ARCHITECTURE.md`.
- Missing or stub `PRINCIPLES.md`.
- Missing mechanical-layer files.

A stale `freshness.json` beyond the declared floor emits a warning with a refresh prompt; it does not block.

Stub detection is explicit. The `[cartography]` block in `.woof/prerequisites.toml` declares `summary_min_chars`; a document below that threshold fails preflight unless it carries front matter proving it is intentionally complete. Setup templates carry a stub marker that must be removed before preflight passes.

The post-commit hook invokes `./scripts/refresh-cartography` as mandatory infrastructure. A missing or non-executable script is already a preflight failure; a script that exits non-zero makes the hook fail loud rather than silently leaving stale mechanical state.

### Per-node loading

The engine loads only the cartography subset relevant to each node. The mapping per stage is defined in `docs/architecture.md`.

## Consequences

- ctags is a hard prerequisite. Preflight verifies the binary is on PATH.
- Operators must author `TARGET-ARCHITECTURE.md` and `PRINCIPLES.md` during setup. The `/woof` umbrella provides the setup flow and references; there is no separate setup skill.
- Woof ships per-language `refresh-cartography` templates. The registry keeps language metadata in `languages/<lang>.toml`; template scripts live under a dedicated template path and are composed by `woof init` into the consumer's `scripts/refresh-cartography` idempotently.
- Tree-sitter is on-demand runtime tooling in the ADR-004 baseline. ADR-009 records the accepted follow-on direction for turning parser-derived facts into an on-disk structural index.
- Mapper subagents are dispatched in parallel by the `/woof` map-codebase flow and write directly to disk; they return confirmation only, keeping the operator context light.
- The design layer and the AS-IS layer (markdown documents) are committed planning state. The mechanical layer is gitignored and regenerated per commit.
- No mid-epic auto-remap of the prose layers. Within an epic, the design and AS-IS (mapper) documents are captured once and stay stable for the duration. The mechanical layer — `tags`, `files.txt`, `freshness.json`, and the ADR-009 structural index — regenerates every commit and so tracks HEAD within an epic; because it is a deterministic function of the commit, any point-in-time state is reproducible for audit without being frozen or committed.
